// turbo-all
1. **获取 Git 状态并自动暂存**
   检查当前 Git 仓库的状态。如果有未跟踪（untracked）或已修改（modified）的文件，则自动将它们添加到暂存区。
   $ git status
   $ git add .

2. **分析变更内容**
   获取暂存区的详细差异，以便生成精准的提交信息。
   $ git diff --cached

3. **生成并执行提交命令**
   根据 `git diff --cached` 的输出分析代码变更。
   
   **必须严格遵守以下规范生成提交消息：**

   ### 规范要求
   - **格式**: `<类型>(<范围>): <简短描述>`
   - **内容**: 随后紧跟可选的详细描述（列表形式）。
   - **语言**: **必须 100% 使用中文**。
   - **语气**: 简洁明了，使用祈使语气（如“修复”、“新增”、“重构”）。

   ### 允许的类型 (type)
   - `feat`: 新功能
   - `fix`: 修复 bug
   - `docs`: 文档更新
   - `style`: 代码格式调整 (不影响逻辑)
   - `refactor`: 重构代码 (无 bug 修复或新功能)
   - `perf`: 性能优化
   - `test`: 测试相关
   - `chore`: 构建系统或工具链相关

   ### 自动执行逻辑
   如果当前是新仓库（Initial Commit），类型使用 `feat`，范围建议为 `init` 或 `project`。
   
   直接执行生成的 `git commit` 命令。例如：
   `git commit -m "feat(init): 初始化项目基础代码
   
   - 包含 ComfyUI 启动脚本和 Docker 配置
   - 添加项目管理相关文档和工作流定义"`

4. **确认提交结果**
   $ git log -n 1
